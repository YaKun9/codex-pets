# 宠物素材授权 / Pet Asset Licensing

本文件适用于各宠物目录中的 `pet.json`、`preview.webp`、`spritesheet.webp` 及其他宠物专属创意素材。仓库根目录的 [MIT License](./LICENSE) **不适用于这些素材**。

This file applies to `pet.json`, `preview.webp`, `spritesheet.webp`, and other pet-specific creative assets inside each pet directory. The repository's root [MIT License](./LICENSE) **does not apply to these assets**.

## 目录级授权 / Per-Directory Licensing

每个宠物目录都必须包含独立的 `LICENSE.md`。该文件应说明投稿者、角色或作品来源、允许的使用方式，以及不包含在授权范围内的第三方权利。对于投稿者能够控制的权利，对应目录的 `LICENSE.md` 是该宠物素材授权范围的最终依据。

Every pet directory must include its own `LICENSE.md`. That file must identify the contributor, the character or source work, permitted uses, and any third-party rights excluded from the permission. For rights the contributor is able to control, the directory-level `LICENSE.md` is the authoritative statement of that pet asset's licensing scope.

## 原创宠物素材 / Original Pet Assets

已取得作者明确授权、且由贡献者拥有必要权利的原创宠物素材，按照对应目录中的 `LICENSE.md` 发布。除非目录另有说明，原创素材采用 [CC BY-NC-SA 4.0](https://creativecommons.org/licenses/by-nc-sa/4.0/)。许可仅覆盖贡献者实际拥有的原创创作部分，不包含任何第三方角色、肖像、商标或其他知识产权。

Original pet assets whose contributors own the necessary rights and have explicitly authorized their use are licensed according to the `LICENSE.md` in each pet directory. Unless otherwise stated, original assets use [CC BY-NC-SA 4.0](https://creativecommons.org/licenses/by-nc-sa/4.0/). The license covers only original elements owned by the contributor and does not include third-party characters, likenesses, trademarks, or other intellectual property.

## 基于第三方角色的素材 / Third-Party Character Fan Assets

基于第三方角色、作品、品牌或人物制作的宠物目录必须使用“素材使用声明”，不能将整个目录直接声明为 MIT 或 CC 授权。现有投稿者已授权本仓库收录其提交内容，并允许用户按照下述范围进行非商业使用，但投稿者只能许可自己实际拥有的创作部分。

Pet directories based on third-party characters, works, brands, or people must use an asset usage notice and must not place the entire directory under MIT or a Creative Commons license. Existing contributors have authorized this repository to include their submissions and allow the non-commercial uses described below, but contributors can license only the original expression they actually own.

在适用法律和相关权利人规则允许的范围内，用户可以按照目录中的 `LICENSE.md`，以非商业目的下载、使用、复制、修改和分享这些素材，包括克隆或 Fork 完整仓库、提交 Pull Request，以及安装到个人 Codex 环境中。分享时必须保留对应目录的 `LICENSE.md` 及其中的来源和署名信息。不得将这些素材用于商业用途、出售、付费分发或转授权。底层角色、名称、设定、肖像、商标及其他相关权利仍归各自权利人所有，本项目与相关权利人不存在隶属、赞助或官方授权关系。

To the extent permitted by applicable law and relevant rights-holder policies, users may download, use, copy, modify, and share these assets for non-commercial purposes under the directory's `LICENSE.md`, including cloning or forking the complete repository, submitting Pull Requests, and installing the assets in a personal Codex environment. Any sharing must retain the applicable directory-level `LICENSE.md` and its source and attribution information. The assets may not be used commercially, sold, distributed for a fee, or sublicensed. Rights in the underlying characters, names, designs, likenesses, trademarks, and related properties remain with their respective rights holders. This project is not affiliated with, sponsored by, or officially endorsed by those rights holders.

## 投稿与权利处理 / Contributions and Rights Requests

新提交的原创宠物素材默认采用 CC BY-NC-SA 4.0；新提交的代码与仓库级文档采用 MIT。所有新宠物 PR 都必须包含目录级 `LICENSE.md`。完整声明与投稿要求见 [CONTRIBUTING.md](./CONTRIBUTING.md)。如权利人认为仓库内容侵犯其权利，请通过 GitHub Issue 联系维护者处理。

New original pet assets use CC BY-NC-SA 4.0 by default, while code and repository-level documentation contributions use MIT. Every new pet Pull Request must include a directory-level `LICENSE.md`. See [CONTRIBUTING.md](./CONTRIBUTING.md) for the complete contributor declaration. Rights holders may contact the maintainers through a GitHub Issue if they believe content should be reviewed or removed.
