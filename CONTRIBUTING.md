# 参与贡献 / Contributing

感谢你帮助 codex-pets 增加新的编码伙伴。具体的宠物目录结构、预览图尺寸和提交流程请参阅 [README.md](./README.md#通过-pull-request-添加宠物) 或 [README_EN.md](./README_EN.md#add-a-pet-via-pull-request)。

Thank you for helping codex-pets welcome new coding companions. For the pet directory structure, preview dimensions, and submission workflow, see [README.md](./README.md#通过-pull-request-添加宠物) or [README_EN.md](./README_EN.md#add-a-pet-via-pull-request).

每个宠物目录都必须包含 `LICENSE.md`。原创宠物使用 CC BY-NC-SA 4.0 模板；基于第三方角色的宠物使用素材使用声明。具体规则见 [ASSETS_LICENSE.md](./ASSETS_LICENSE.md)。

Every pet directory must include a `LICENSE.md`. Original pets use the CC BY-NC-SA 4.0 template, while pets based on third-party characters use an asset usage notice. See [ASSETS_LICENSE.md](./ASSETS_LICENSE.md) for the governing rules.

## PR 中需要说明 / Required PR Information

宠物素材 PR 请在说明中包含：

- 角色或作品来源，以及属于原创还是第三方角色同人作品；
- 素材作者与 GitHub 用户名；
- 你拥有相关权利或已取得授权的确认；
- 原创素材采用 CC BY-NC-SA 4.0 的确认。
- 宠物目录中包含与素材类型相符的 `LICENSE.md`。

Pet asset Pull Requests must include:

- the character or work source, and whether the pet is original or a third-party character fan work;
- the asset creator and GitHub username;
- confirmation that you own the necessary rights or have permission;
- confirmation that original assets are contributed under CC BY-NC-SA 4.0.
- a `LICENSE.md` in the pet directory that matches the asset type.

现有素材的授权状态以 [ASSETS_LICENSE.md](./ASSETS_LICENSE.md) 和各目录中的明确声明为准。

Existing assets are governed by [ASSETS_LICENSE.md](./ASSETS_LICENSE.md) and the explicit notices in their applicable directories.
